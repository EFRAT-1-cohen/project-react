## תיאור הפרויקט

מערכת Help Desk מקיפה המאפשרת ניהול יעיל של פניות שירות עם תמיכה בשלושה תפקידי משתמש: מנהל מערכת (Admin), נציג שירות (Agent) ולקוח (Customer). המערכת כוללת ממשק משתמש ידידותי בעברית עם תמיכה מלאה ב-RTL.

## ✨ תכונות עיקריות

### ניהול טיקטים
- יצירה, צפייה ועריכה של טיקטים
- מערכת סטטוסים ועדיפויות
- הקצאת טיקטים לנציגי שירות
- מערכת הערות ותגובות
- סינון וחיפוש מתקדם

### ניהול משתמשים (Admin)
- יצירה ועריכה של משתמשים
- הקצאת תפקידים ורמות גישה
- ניהול סטטוסים פעילים/לא פעילים
- צפייה בפעילות משתמשים

### דשבורד אישי
- סטטיסטיקות מותאמות לפי תפקיד
- גרפים ודוחות ויזואליים (Recharts)
- מעקב אחר ביצועים

### אימות והרשאות
- מערכת התחברות מאובטחת
- ניהול הרשאות לפי תפקידים
- שמירת מצב התחברות (JWT)

- **Material-UI (MUI) 7.3** - ספריית קומפוננטות UI
- **React Router DOM 7.11** - ניווט ונתוב
- **Emotion** - CSS-in-JS
- **React Hook Form 7.69** - ניהול טפסים
- **Axios 1.13** - בקשות HTTP
- **Recharts 3.6** - גרפים ודוחות

## 🚀 התקנה

1. שכפל את הפרויקט:
```bash

cd new-project-main/helpdesk-client
```

2. התקן את התלויות:

## 👥 תפקידי משתמשים

### Admin (מנהל מערכת)
- ניהול מלא של משתמשים
- גישה לכל הטיקטים
- צפייה בדוחות וסטטיסטיקות
- ניהול הגדרות מערכת

### Agent (נציג שירות)
- צפייה בטיקטים המוקצים
- עדכון סטטוס וטיפול בטיקטים
- הוספת תגובות והערות
- צפייה בטיקטים פתוחים

### Customer (לקוח)
- יצירת טיקטים חדשים
- צפייה בטיקטים אישיים
- מעקב אחר סטטוס פניות
- הוספת תגובות לטיקטים שלו

## 🔌 API ושירותים

המערכת עובדת עם שירותי API הבאים:

### Authentication Service (`authService.ts`)
- התחברות והרשמה
- ניהול טוקנים
- בדיקת הרשאות

### Tickets Service (`ticketsService.ts`)
- CRUD פעולות על טיקטים
- ניהול הערות
- סינון וחיפוש
- עדכון סטטוסים ועדיפויות

### Users Service (`usersService.ts`)
- ניהול משתמשים
- הקצאת תפקידים
- עדכון פרטים
